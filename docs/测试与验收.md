本文件记录 AICR Reviewer 项目的常用工具脚本、验收流程及注意事项。

| 脚本 | 说明 |
|------|------|

## gitlab_group_clone.ps1

### 功能

从 `~/.opencodereview/config.json` 读取 GitLab 配置，支持三种模式：

1. **GroupSummary**：列出当前 token 可访问的 Group 清单及其项目数量，导出 CSV/JSON 汇总报告（默认不过滤组内权限级别；可选 `-MinAccessLevel`）。
2. **BatchClone**：针对一个或多个指定 Group 按批次克隆项目到本地目录（默认 `D:\agit`）。
3. **GlobalDayClone**：基于稳定清单按日额度切片克隆（默认每天 50 个），用于全站多日节奏。

### 指定 Group

| 方式 | 示例 |
|------|------|
| 单个 Group | `-TargetGroupPath aulton-ms-basics` |
| 多个 Group（数组） | `-TargetGroupPath @('aulton-ms-basics','aulton-ms-finance')` |
| 多个 Group（逗号） | `-TargetGroupPath aulton-ms-basics,aulton-ms-finance` |
| 从文件读取 | `-TargetGroupListFile D:\agit\groups.txt`（每行一个 path，`#` 为注释） |

> 指定多个 Group 时，脚本会**依次**处理每个 Group；`-BatchIndex` / `-BatchSize` 对每个 Group **分别**生效。

### 常用参数（BatchClone）

| 参数 | 必填 | 默认值 | 说明 |
|------|------|--------|------|
| `-TargetGroupPath` | 是（BatchClone） | `aulton-ms-basics` | 要克隆的 Group，可多个 |
| `-BatchSize` | 否 | `20` | 每批项目数；`0` = 只列出不克隆 |
| `-BatchIndex` | 否 | `0` | 批次序号，从 0 递增 |
| `-CheckoutRelease` | 否 | `$true` | 克隆/更新后切到 release 分支 |
| `-ReleaseBranch` | 否 | `release` | 目标分支名 |
| `-SkipExisting` | 否 | `$true` | 本地已有 `.git` 则跳过 clone，不重复下载 |
| `-UpdateExisting` | 否 | 关闭 | 已存在仓库执行 `git pull`（优先于 SkipExisting） |
| `-IncludeSubgroups` | 否 | 开启 | 是否包含子组项目 |

关闭 release 切换：`-CheckoutRelease:$false`。自定义分支：`-ReleaseBranch master`。

### GlobalDayClone（全站多日克隆）

先生成**稳定清单**（默认 `D:\agit\_reports\global_project_manifest.json`），再每天递增 `-DayIndex`。后续天**不要**随意 `-RefreshManifest`，否则切片会漂移。

```powershell
# Day0：盘点 Group（只读，可选）
.\scripts\gitlab_group_clone.ps1

# 生成/刷新清单并只查看全部项目（不 clone）
.\scripts\gitlab_group_clone.ps1 -Mode GlobalDayClone -DailyLimit 0 -RefreshManifest

# 第 1 天：清单中第 1–50 个
.\scripts\gitlab_group_clone.ps1 -Mode GlobalDayClone -DayIndex 0 -DailyLimit 50

# 第 2 天（复用同一清单）
.\scripts\gitlab_group_clone.ps1 -Mode GlobalDayClone -DayIndex 1 -DailyLimit 50
```

| 参数 | 默认 | 说明 |
|------|------|------|
| `-DayIndex` | `0` | 第几天（从 0 起）；超出范围时提示「已全部处理完」 |
| `-DailyLimit` | `50` | 每天克隆上限；`0` = 只列清单不 clone |
| `-ManifestPath` | `...\global_project_manifest.json` | 稳定清单路径 |
| `-RefreshManifest` | 关 | 强制重拉 API 覆盖清单 |
| `-EmbedTokenInCloneUrl` | `$true` | HTTPS clone 临时嵌入 PAT（报告中脱敏） |
| `-SkipExisting` | `$true` | 同一天可重跑；已存在跳过且**不**再 checkout |
| `-CheckoutRelease` | `$true` | 新 clone/更新 后切 release |

项目发现：`/projects?membership=true` 与 `/projects?all_available=true` 合并按 `id` 去重，再按 `path_with_namespace` 排序写入清单。日报告：`D:\agit\_reports\global_day_clone_<DayIndex>_*.json`。有失败时退出码为 `1`。

前置：配置中的 `gitlab.api_token` 需具备 `read_api`/`api`；默认嵌入 token clone 时还需 token 含 `read_repository`（或等价拉代码权限）。

### 配置示例

在 Windows 用户目录创建 `~/.opencodereview/config.json`：

```json
{
  "gitlab": {
    "url": "https://gitlab.aulton.com",
    "api_token": "YOUR_GITLAB_PAT"
  }
}
```

> `api_token` 也可写成 `auth_token`；脚本兼容旧版顶层字段 `gitlab_api_token`。

### 使用示例

```powershell
# 1. 汇总所有 Group 及项目数（先执行，用于规划分批）
cd aicr-reviewer
.\scripts\gitlab_group_clone.ps1

# 2. 仅列出目标组的项目，不克隆（确认范围）
.\scripts\gitlab_group_clone.ps1 -Mode BatchClone -TargetGroupPath aulton-ms-basics -BatchSize 0

# 3. 克隆单个 Group（使用默认 BatchSize=20、BatchIndex=0，并自动 checkout release）
.\scripts\gitlab_group_clone.ps1 -Mode BatchClone -TargetGroupPath aulton-ms-finance

# 3a. 显式指定批次（项目数 >20 时需递增 BatchIndex）
.\scripts\gitlab_group_clone.ps1 -Mode BatchClone -TargetGroupPath aulton-ms-basics -BatchSize 20 -BatchIndex 0

# 3a2. 不切 release 分支
.\scripts\gitlab_group_clone.ps1 -Mode BatchClone -TargetGroupPath aulton-ms-finance -CheckoutRelease:$false

# 3b. 克隆多个 Group（各自第 0 批）
.\scripts\gitlab_group_clone.ps1 -Mode BatchClone -TargetGroupPath aulton-ms-basics,aulton-ms-finance -BatchSize 20 -BatchIndex 0

# 3c. 从文件读取 Group 列表
# groups.txt 示例：
#   aulton-ms-basics
#   aulton-ms-finance
.\scripts\gitlab_group_clone.ps1 -Mode BatchClone -TargetGroupListFile D:\agit\groups.txt -BatchSize 20 -BatchIndex 0

# 4. 已存在仓库执行 pull 更新（默认跳过）
.\scripts\gitlab_group_clone.ps1 -Mode BatchClone -TargetGroupPath aulton-ms-basics -BatchSize 20 -BatchIndex 0 -UpdateExisting
```

### 输出

- 汇总报告：`D:\agit\_reports\group_project_summary_*.csv` / `*.json`
- 批次报告：`D:\agit\_reports\batch_clone_<group>_<batch_index>_*.json`
- 日报告（GlobalDayClone）：`D:\agit\_reports\global_day_clone_<DayIndex>_*.json`
- 仓库目录：`D:\agit\<group_path>\<project_name>`

### 分批执行建议

1. 先运行 `GroupSummary` 粗看 Group 规模；**全站真实项目数与多日切片以 `global_project_manifest.json` 为准**。
2. **全站按天 50 个**：`GlobalDayClone`，首日生成清单后只改 `-DayIndex`；失败看日报告 `failed` 并同日重跑（exit code 1）。
3. **仅某几个 Group**：仍用 `BatchClone`；每个 Group 从 `BatchIndex = 0` 递增。
4. 不要用「多 Group + 同一 BatchIndex」凑每日额度；也不要用 `-RefreshManifest` 打断已开始的多日序列（除非有意重开）。

### 跳过已 clone 仓库

**默认已开启**（`-SkipExisting:$true`）：若本地路径下已有 `.git` 目录，输出 `[SKIP] 已存在` 并跳过 clone，不会重复下载。

本地路径规则：`D:\agit\<group_path>\<project_name>\`（与 GitLab `path_with_namespace` 对应）。

| 场景 | 行为 |
|------|------|
| 目录有 `.git`，默认参数 | **跳过** clone |
| 目录有 `.git`，加 `-UpdateExisting` | **pull** 更新，不重新 clone |
| 目录非空但无 `.git` | **警告并跳过**（避免覆盖失败残留目录） |
| 强制不跳过（极少用） | `-SkipExisting:$false`，已有仓库时会报错 |

## gitlab_batch_checkout.ps1

### 功能

对已 clone 到本地的工程**批量 checkout 到指定分支**；若远端存在 `origin/<分支>` 则 **pull 最新代码**。

与 `gitlab_group_clone.ps1` 共用目录布局：`D:\agit\<group_path>\<project_name>\`。

### 执行流程

1. **预检（先检查后操作）**：逐个检查目标路径是否为 git 仓库、工作区是否干净；
2. **任一失败则整批中断**，不修改任何仓库；
3. 全部通过后：`fetch` → `checkout` →（远端有分支时）`pull`。

### 参数

| 参数 | 必填 | 默认 | 说明 |
|------|------|------|------|
| `-TargetGroupPath` | 是 | — | Group 路径，如 `aulton-ms-finance` |
| `-ProjectNames` | 二选一 | — | 项目名，可多个或逗号分隔 |
| `-ProjectListFile` | 二选一 | — | 每行一个项目名，`#` 为注释 |
| `-Branch` | 否 | `release` | 目标分支 |
| `-DownloadRoot` | 否 | `D:\agit` | 本地根目录 |

### 使用示例

```powershell
cd aicr-reviewer

# finance 组下两个工程 → release 并 pull
.\scripts\gitlab_batch_checkout.ps1 `
  -TargetGroupPath aulton-ms-finance `
  -ProjectNames aulton-ms-pay,aulton-ms-split-bill

# 自定义分支
.\scripts\gitlab_batch_checkout.ps1 `
  -TargetGroupPath aulton-ms-finance `
  -ProjectNames aulton-ms-pay `
  -Branch master

# 从文件读取项目列表（projects.txt 每行一个项目名）
.\scripts\gitlab_batch_checkout.ps1 `
  -TargetGroupPath aulton-ms-finance `
  -ProjectListFile D:\agit\finance_projects.txt
```

### 未提交代码

预检使用 `git status --porcelain`（含未跟踪文件）。若任一工程有变更，脚本退出码 `1` 并提示先 commit / stash / 还原。

### 输出

- 报告：`D:\agit\_reports\batch_checkout_<group>_<timestamp>.json`
